Regula

Regula is a powerful and easy-to-use client-side validation-library written in JavaScript. It features validation constraints that can be directly attached to elements via markup, not unlike HTML5 validation constraints.

Features

  • Annotation-based - Validation semantics are conveyed by annotations (using the data-constraints HTML5 data- attribute). This means that it is not necessary to resort to JavaScript (although it is possible if required).
  • Self-contained - Regula does not depend on any external libraries.
  • Easy - Although Regula sports complex-features, it is easy to understand and use.
  • Customizable - Regula is built with custom validation in mind. It is trivial to define and use custom validation-constraints to handle complex validation-scenarios.
  • Powerful - Regula supports many powerful-features such as compound constraints (constraints that contain other constraints), validation groups (segmented validation), customizable error-messages with parameter interpolation, asynchronous validation, and HTML5-validation support (if natively supported, or provided by a shim).

Quick Examples

Marking fields as "required"

  • Using @Required:
      <input type="text"
         name="username" 
         id="username" 
         data-constraints="@Required" />
  • Using @Required with custom messages and labels:
      <input type="text" 
         name="firstName"
         id="firstName"
         data-constraints='@Required(label="First Name", message="{label} is required.")' />
  • Using native HTML5 required constraint (Regula will recognize if provided natively or via an HTML5 shim):
      <input type="text" name="lastName" id="lastName" required="true" />
  • Using Regula's "wrapped" HTML5 constraint, which let you provide custom messages and labels:
      <input type="text"
         name="company"
         id="company"
         data-constraints='@HTML5Required(label="Company", message="{label} is required.")' />

Examples of other usage-scenarios and constraints

  • Using @Max and @Min:
      <input type="text"
         name="powerLevel"
         id="powerLevel"
         data-constraints='@Max(value=9000, label="Power Level", message="{label} cannot be over {value}!")' />

  <input type="text"
         name="age"
         id="age"
         data-constraints='@Min(value=21, message="You must be over {value} to buy beer!")' />
  • Pattern matching using @Pattern:
      <input type="text"
         name="ssn"
         id="ssn"
         data-constraints='@Pattern(regex=/[0-9]{3}-[0-9]{2}-[0-9]{4}/, message="Must be a valid social-security number")' />
  • Ensuring that passwords match:
      <form id="myForm" name="myForm"
        ...
        data-constraints='@PasswordsMatch(field1="password", field2="confirmationPassword")'>

        Enter a password: <input id="password" name="password" type="password" />
        Confirm password: <input id="confirmationPassword" name="confirmationPassword" type="password" />
  </form>
  • Validation groups:
      <input type="text" id="firstName" name="firstName"
         data-constraints="@Required(groups=[Name, All])" />
  <input type="text" id="lastName" name="lastName"
         data-constraints="@Required(groups=[Name, All])" />

  <input type="text" id="street1" name="street1"
         data-constraints="@Required(groups=[Address, All])" />
  <input type="text" id="street2" name="street2"
         data-constraints="@Required(groups=[Address, All])" />
  <input type="text" id="city" name="city"
         data-constraints="@Required(groups=[Address, All])" />
  <input type="text" id="state" name="state"
         data-constraints="@Required(groups=[Address, All])" />
  <input type="text" id="zip" name="zip"
         data-constraints="@Required(groups=[Address, All])" />

Custom constraints

  • Defining and using a custom constraint:
    Javascript
      regula.custom({
      name: "RegistryNumber",
      formSpecific: false,
      defaultMessage: "{label} is an invalid registry number.",
      validator: function(params, validator, callback) {
          return /(NCC|NX)-[0-9]+(-[A-Z])?/.test(this.value);
      }
  });
    HTML
      <input type="text" id="registryNumber" name="firstName"
         data-constraints="@RegistryNumber" />
  • Defining and using a custom constraint with a parameter:
    Javascript
      regula.custom({
      name: "DivisibleBy",
      formSpecific: false,
      params: ["divisor"],
      defaultMessage: "{label} must be divisible by {divisor}.",
      validator: function(params, validator, callback) {
          var divisor = parseInt(params.divisor, 10);
          var value = parseInt(this.value, 10);

          return (value % divisor) === 0;
      }
  });
    HTML
      <input type="text" id="number" name="number"
         data-constraints='@DivisibleBy(divisor="3", label="The number")' />
  • Conditional validation through a custom constraint:
    Javascript
      regula.custom({
      name: "ConditionalRequired",
      formSpecific: false,
      defaultMessage: "{label} is required.",
      validator: function(params, validator, callback) {
          var valid = true;

          if(document.getElementById("myCheckbox").checked) {
              validator = validator.required(this, params);
          }

          return valid;
      }
  });
    HTML
      <input type="checkbox" id="over21", name="over21" /> I am over 21!
  <input type="text" id="nickname" name="nickname"
         data-constraints='@ConditionalRequired(label="The nickname")' />
  • Defining and using a compound constraint:
    Javascript
      //A compound constraint that consists of @Range and @Digits. Ensures that a number is
  //between max and min and also has two fractional digits.
  regula.compound({
      name: "RangedDecimalsAtTwoPlaces"
      params: ["max", "min"],
      constraints: [
          {
              constraintType: regula.Constraint.Range
          }, 
          {
              constraintType: regula.Constraint.Digits,
              params: {
                  integer: 10,
                  fraction: 2
              }
          }
      ]
  });
    HTML
      <input type="text" id="price" name="price"
         data-constraints='@RangedDecimalsAtTwoPlaces(label="The price", min=10, max=42)' />

Performing validation

  • Validating everything and displaying any errors:
      var constraintViolations = regula.validate();
  for(var i = 0; i < constraintViolations.length; i++) {
      var constraintViolation = constraintViolations[i];

      for(var j = 0; j < constraintViolation.failingElements.length; j++) {
          //Assuming the browser supports 'setCustomValidity'; otherwise you will have to render
          //the errors in some other manner
          constraintViolation.failingElements[j].setCustomValidity(constraintViolation.message);
      }
  }
  • Validating specific groups:
      var constraintViolations = regula.validate({
      groups: [regula.Group.Address]
  });
  • Validating specific elements:
      var constraintViolations = regula.validate({
      elements: [document.getElementById("element1"), docuent.getElementById("element2")]
  });
  • Validating asynchronous constraints:
      regula.validate(function(constraintViolations) {
      for(var i = 0; i < constraintViolations.length; i++) {
          var constraintViolation = constraintViolations[i];

          for(var j = 0; j < constraintViolation.failingElements.length; j++) {
              //Assuming the browser supports 'setCustomValidity'; otherwise you will have to render
              //the errors in some other manner
              constraintViolation.failingElements[j].setCustomValidity(constraintViolation.message);
          }
      }
  });
